## Challenge
There a many CSS libraries and frameworks out there, so why build another?

Mainly because I wanted to learn more about Sass, LESS and what actually goes into building something like Bootstrap, but was not as bloated and large as Bootstrap. I also wanted to make something that was really simple and easy to use, something that would make the lives of my friends and colleagues, in fact anyone who wanted to use it much easier when developing sites.

## Solution
I started work on an initial version of would become Doughnut while building a Pilates website. It was a really simple code base and only had some base items styled, like buttons and inputs. Version 1 also included a grid system which was similar to all the others out there, it just used a different naming scheme  `<div class="eight column">` which worked well and was easy to understand.

I had a few people start using this initial concept which allowed me to gain some valuable feedback. With this feedback i was able to start to think about how to improve the library and really push it forward. I spent some time looking into ways that Sass can help improve the code base, and then started working on these minor modifications.

Work soon began on Doughnut. Doughnut introduced the new grid system, which only generates the code for the columns that are used. You are also able to name the columns anything you want, this opens up more freedom than other library's or frameworks. - Check out the [Doughnut site](http://doughnut.r3id.io) for more info and examples.

## Result
Doughnut CSS has allowed me to build great looking sites, without the complexities and size of the other library's and frameworks offer. It has given me the freedom to develop responsive sites quicker, which work across multiple platforms and screens.
